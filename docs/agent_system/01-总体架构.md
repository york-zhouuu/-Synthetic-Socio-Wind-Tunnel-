# Agent 系统设计：总体架构

## 设计目标

在 synthetic_socio_wind_tunnel 现有基础设施（Atlas/Ledger/Simulation/Perception）之上，构建自主 Agent 系统，支持：
- LLM 驱动的自主决策（**所有 agent 都用 LLM，无纯规则 agent**）
- 三层记忆（Stanford 式：记忆流 + 检索 + 反思）
- 关系网络追踪（弱关系形成、信任演变）
- **目标规模：1000 个 agent**（动态模型梯度分配）
- 可被 Policy Hack 干预并产生可观测的行为变化

---

## 核心设计决策

### 决策循环：计划制 LLM

**为什么选计划制而不是全 LLM 或纯规则：**

| 方案 | 1000 agent × 1 天 (100 tick) 的 LLM 调用量 | 行为真实度 | 可观测性 |
|------|----------------------------------------------|-----------|---------|
| 全 LLM | 100,000 次 | 最高 | 高但贵 |
| **计划制 LLM** | **~2,000-5,000 次** | **高** | **最好** |
| 规则 + LLM | ~500 次 | 中 | 中 |

**计划制 LLM 的工作方式：**

```
每个模拟日开始时:
    for agent in all_agents:
        plan = LLM.generate_daily_plan(agent.profile, agent.memory, current_context)
        # 产出: [
        #   {time: "7:00", action: "leave_home", destination: "bus_stop"},
        #   {time: "7:30", action: "commute", destination: "office_building"},
        #   {time: "12:00", action: "lunch", destination: "sunrise_cafe"},
        #   ...
        # ]

每个 tick:
    for agent in all_agents:
        current_step = agent.plan.current()

        # 检查是否有打断事件
        interrupts = check_interrupts(agent)
        if interrupts:
            # 只在被打断时才重新调 LLM
            new_plan = LLM.replan(agent, interrupts, remaining_plan)
            agent.plan = new_plan
        else:
            # 按计划执行，不调 LLM
            execute(current_step)
```

**关键洞察：计划的"偏离"正是实验要观测的核心指标。**

agent 本来计划早上通勤 A→B，但收到一条 Policy Hack 信息（"附近酒吧今晚试饮"）后，下班改去了 C。这个偏离：
- 可以量化（轨迹弯曲度）
- 可以归因（因为什么信息导致了改变）
- 可以对比（干预组 vs 对照组的偏离程度）

### 模型梯度分配（替代固定三层）

**原则：不存在"纯规则 agent"。所有 1000 个 agent 都用 LLM，但模型等级动态分配。**

#### 10 个 Protagonist — 固定高配

```
模型: Claude Sonnet 级别 (固定不变)
记忆: 完整三层 (事件流 + 摘要 + 反思)
社交: 完整多轮对话
叙事: 产出罗生门视角的第一人称叙事
用途: 深度观察对象，实验报告的主角
```

#### 990 个 Dynamic Agent — 三层叠加的模型分配

模型等级不是固定的，而是由三层策略叠加决定：

```
┌─────────────────────────────────────────────────────────────┐
│  Layer 1: 基础梯度 (初始化时计算，每日更新)                    │
│                                                             │
│  依据: 与 Protagonist 的社交距离                              │
│                                                             │
│  距离 1 (直接关系: 邻居/同事/朋友)                             │
│    → 基础模型 = Sonnet-mini 级别                              │
│    → 记忆 = 事件流 + 摘要                                    │
│                                                             │
│  距离 2 (间接关系: 朋友的朋友)                                 │
│    → 基础模型 = Haiku 级别                                   │
│    → 记忆 = 事件流 + 简短摘要                                 │
│                                                             │
│  距离 3+ (无直接关系)                                         │
│    → 基础模型 = Haiku 级别 (最便宜)                           │
│    → 记忆 = 事件流                                           │
├─────────────────────────────────────────────────────────────┤
│  Layer 2: 场景升级 (每个 tick 动态判断)                        │
│                                                             │
│  触发升级的场景:                                              │
│  ├─ 与 Protagonist 在同一 location → 升 2 级                 │
│  ├─ 被 Policy Hack 直接触达 → 升 1 级                        │
│  ├─ 进入社交对话 → 升 1 级 (对话期间)                         │
│  ├─ 做出重要决策 (改变计划) → 升 1 级 (本次决策)               │
│  └─ 进入实验热点区域 → 升 1 级                                │
│                                                             │
│  降级时机:                                                    │
│  ├─ 对话结束后 → 降回基础等级                                  │
│  ├─ 离开热点区域后 → 降回基础等级                              │
│  └─ 但: 重要记忆保留，不因降级丢失                             │
├─────────────────────────────────────────────────────────────┤
│  Layer 3: 全局预算调度 (实时)                                  │
│                                                             │
│  每个 tick 有一个 LLM 调用预算上限                             │
│  如果同时有太多 agent 需要升级:                                │
│  ├─ 按优先级排序: Protagonist 互动 > Policy Hack > 普通社交    │
│  ├─ 预算内的升级，预算外的维持基础等级                          │
│  └─ 记录"应升级但未升级"的情况，供分析                         │
└─────────────────────────────────────────────────────────────┘
```

#### 模型等级定义

```
Level 4: Sonnet 级  — Protagonist 固定使用
Level 3: Sonnet-mini / 4o-mini 高配  — 升级后的近距离 agent
Level 2: Haiku 级  — 大部分 agent 的日常
Level 1: Haiku 级 (精简 prompt)  — 远距离 agent 的日常
```

注意：Level 1 和 Level 2 可能用同一个模型 (Haiku)，区别在于 prompt 的丰富程度和 context window 的大小。Level 1 的 prompt 更短、记忆更少，所以更便宜。

#### 成本估算（更新版）

| 场景 | 调用量 | 模型分布 | 预估成本 |
|------|--------|---------|---------|
| 日计划生成 | 1000 次 | 10 Sonnet + 990 Haiku | ~$1.5 |
| 重规划 | ~100-200 次 | 混合 | ~$0.5 |
| 社交对话 | ~200-400 次 | 按场景升级 | ~$1.0 |
| 记忆摘要 | ~100 次 | 10 Sonnet + ~90 Haiku | ~$0.3 |
| **每模拟日总计** | **~1,500-1,700** | | **~$3-5** |

14 天实验 ≈ $42-70，完全可控。

       │                                              │

## 模拟时钟设计

| 概念 | 设计 |
|------|------|
| **1 tick** | = 模拟中的 5 分钟 |
| **1 模拟日** | = 288 ticks (24h × 12 ticks/h) |
| **活跃时段** | 7:00-23:00 = 192 ticks (只在此期间处理决策) |
| **计划粒度** | 每个计划条目约 30 分钟 - 2 小时 |
| **一个实验** | 通常跑 7-14 模拟日 |

```
模拟日结构:

00:00 ─────── 07:00 ─────── 12:00 ─────── 18:00 ─────── 23:00 ── 00:00
  [  睡眠/跳过  ] [ 上午活动  ] [  午间活动  ] [  晚间活动  ] [睡眠]
                  │            │              │
                  └── 生成日计划 ──────────────┘
                      (1次 LLM 调用/agent)
```

## 性格系统

**性格数据由外部调研文档输入，不在本文档定义具体参数。**

AgentProfile 中的性格字段将基于用户提供的调研数据来设计，确保：
- 性格维度基于真实的社会学/心理学框架
- 每个 agent 的性格描述足够具体，能让 LLM 产生差异化行为
- 1000 个 agent 的性格分布符合真实人口统计

性格如何影响 LLM 行为的具体设计，待调研文档补充后确定。
