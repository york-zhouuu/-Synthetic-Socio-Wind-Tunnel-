# Agent 核心模块设计

## 1. AgentProfile — 性格与身份

**文件**: `synthetic_socio_wind_tunnel/agent/profile.py`

AgentProfile 定义 agent "是谁"，在整个模拟过程中基本不变。

### 数据模型

```python
class AgentProfile(BaseModel):
    """Agent 的静态身份和性格定义。模拟期间不变。"""

    # === 身份 ===
    agent_id: str                    # 唯一标识
    name: str                        # "Emma Chen"
    age: int                         # 28
    occupation: str                  # "software_engineer"
    household: str                   # "single" / "couple" / "family_with_kids"

    # === 居住 ===
    home_location: str               # 家的 location_id (初始值，可被 Life Pattern 更新)

    # === 性格特征 ===
    # 具体维度和数据来源由外部调研文档定义
    # 这里只定义数据结构，不预设维度
    personality_traits: dict[str, float] = {}    # {"extroversion": 0.3, "curiosity": 0.8, ...}
    personality_description: str = ""            # 具体行为描述 (给 LLM 用)
    # 例: "你不喜欢主动搭话，路上遇到邻居通常只点头微笑。
    #      但你对新事物很好奇，看到有趣的东西会停下来观察。"

    # === 社交偏好 ===
    preferred_social_size: int       # 偏好的社交群体大小 (1=独处, 5=派对)
    interests: list[str]             # 兴趣标签: ["coffee", "reading", "chess", "gardening"]
    languages: list[str]             # 语言: ["english", "mandarin"]

    # === 日常习惯 ===
    wake_time: str                   # "7:00"
    sleep_time: str                  # "23:00"
    # 注: work_location, commute_route, routine_places 不再硬编码在 Profile 中
    # 它们作为 Life Pattern (记忆 Layer 4) 存在，可以被经历更新

    # === LLM 配置 ===
    is_protagonist: bool = False     # 是否为 10 个主角之一
    base_model: str = "claude-haiku-4-5-20251001"  # 基础模型 (会被动态梯度覆盖)

    model_config = {"frozen": True}
```

### 性格如何影响行为

**性格数据由外部调研文档输入**，不在代码中预设固定维度。

`personality_traits` 是一个灵活的 dict，可以包含任意维度。`personality_description` 是自然语言描述，直接嵌入 LLM prompt——这比数值更能让 LLM 产生差异化行为。

**在 Planner 的 LLM prompt 中体现：**
```
你是 Emma Chen，28岁软件工程师。

{agent.personality_description}

（以上性格描述基于调研数据，请据此生成你今天的日程。）
```

性格影响行为的具体机制（比如什么情况下重规划、什么时候主动社交）将在 Planner 和 SocialGraph 中通过 `personality_traits` 的数值来控制，但具体用哪些 trait 维度，取决于调研文档的定义。

## 2. Memory — 四层记忆系统

**文件**: `synthetic_socio_wind_tunnel/agent/memory.py`

参考 Stanford Generative Agents 的架构，增加 Life Pattern 层，为 1000 agent 规模做了适配。

### 架构

```
┌─────────────────────────────────────────────────┐
│          Layer 4: Life Pattern (生活模式)          │
│                                                  │
│  "我在 office_building 上班，周一到周五"            │
│  "午饭通常去 sunrise_cafe，偶尔去 fresh_mart"       │
│  "周末早上去 central_park 跑步"                    │
│  "通勤路线: home → elm_st → main_st → office"     │
│  "最近发现了一家新的面包店，开始常去"                │
│                                                  │
│  特征:                                            │
│  - 超长期，极少变化                                │
│  - 是 Planner 生成日计划的基础框架                  │
│  - 可被重大事件/持续行为更新                        │
│  - 初始化时由 agent 设定文档 + LLM 生成             │
│  - 更新频率: 每 3-7 模拟日检查一次                  │
├─────────────────────────────────────────────────┤
│          Layer 3: Reflections (反思)              │
│  "我发现我最近经常在咖啡馆遇到那个下棋的老人"        │
│  定期由 LLM 从 Layer 2 中归纳生成                  │
│  频率: 每模拟日 1 次 (只对 Protagonist)             │
├─────────────────────────────────────────────────┤
│          Layer 2: Summaries (摘要)                │
│  "今天上午去了咖啡馆，遇到了 3 个人，                │
│   和 Bob 聊了天气，发现公园在办活动"                │
│  每段活动结束后由 LLM 压缩生成                     │
│  频率: 每模拟日 3-5 次                             │
├─────────────────────────────────────────────────┤
│          Layer 1: Event Stream (事件流)            │
│  [10:05] arrived_at sunrise_cafe                  │
│  [10:07] noticed Bob sitting at corner            │
│  [10:08] Bob said "nice weather today"            │
│  [10:15] heard announcement about park event      │
│  原始事件流，直接从 WorldEvent 转化                 │
│  自动记录，不需要 LLM                              │
└─────────────────────────────────────────────────┘
```

**四层的关系：**

```
Layer 4 (Life Pattern)  ──→  Planner 的基础输入
    ↑ 持续行为触发更新           "我通常在哪上班、去哪吃饭"
    │
Layer 3 (Reflections)   ──→  Planner 的情境补充
    ↑ LLM 归纳                 "最近有什么规律/变化"
    │
Layer 2 (Summaries)     ──→  Planner 的近期参考
    ↑ LLM 压缩                 "昨天发生了什么"
    │
Layer 1 (Events)        ──→  实时感知记录
                              "刚才发生了什么"
```

### 数据模型

```python
class LifePatternEntry(BaseModel):

        "Emma Chen 是一名 28 岁的软件工程师，住在 maple_apartments。

        2. 重要性: importance 高的优先
```

### 不用 Embedding 的检索策略

Stanford 原版用 embedding 做语义检索，但对 1000 agent 来说成本和复杂度太高。简化方案：

```python
def retrieve(self, query: str, k: int = 5) -> list[MemoryEntry]:
    """基于规则的记忆检索，不需要 embedding。"""
    scored = []
    now = self.current_time
    query_words = set(query.lower().split())

    for memory in self.all_memories():
        score = 0.0

        # 时间衰减 (指数衰减，半衰期 = 3 天)
        age_hours = (now - memory.timestamp).total_seconds() / 3600
        recency = 0.5 ** (age_hours / 72)
        score += recency * 1.0

        # 重要性
        score += memory.importance * 2.0

        # 关键词匹配
        memory_words = set(memory.content.lower().split())
        overlap = len(query_words & memory_words)
        score += overlap * 0.5

        # 类型加权
        type_weight = {"reflection": 1.5, "summary": 1.0, "event": 0.3}
        score *= type_weight.get(memory.memory_type, 1.0)

        # 相关 agent 匹配
        for agent_id in memory.related_agents:
            if agent_id in query:
                score += 1.0

        scored.append((score, memory))

    scored.sort(key=lambda x: x[0], reverse=True)
    return [m for _, m in scored[:k]]
```

### 分层记忆策略

所有 agent 都有记忆，但深度随模型等级动态调整：

| 模型等级 | Layer 1 (事件) | Layer 2 (摘要) | Layer 3 (反思) |
|---------|---------------|---------------|---------------|
| Protagonist (固定) | 保留全部 | 每段活动后生成 | 每天生成 |
| 升级中 (临时高配) | 保留全部 | 升级期间生成 | 不生成 |
| 基础等级 | 保留最近 100 条 | 每天生成 1 条 | 不生成 |

**关键：降级不丢失记忆。** agent 在升级期间获得的重要记忆（如从 Protagonist 那里听到的信息）会永久保留在 Layer 1 事件流中，即使模型降回 Haiku 也能在后续日计划生成时被检索到。

### 计划生成 Prompt

Planner 的核心输入是 **Life Pattern (Layer 4)**，而不是 AgentProfile 中硬编码的地点。

```python
def build_plan_prompt(agent: AgentProfile, memory: AgentMemory, context: dict) -> str:
    """构建日计划生成的 LLM prompt。prompt 丰富度随模型等级变化。"""

    # 从 Life Pattern 获取今天的生活框架
    daily_framework = memory.get_daily_framework(context["day_of_week"])
    # → [LifePatternEntry("我在 TechCorp 上班，周一到周五 9:00-18:00"),
    #    LifePatternEntry("午饭通常去 sunrise_cafe"),
    #    LifePatternEntry("周末早上去 central_park 跑步"), ...]

    # 基础 prompt (所有等级都有)
    prompt = f"""
你是 {agent.name}，{agent.age}岁，职业是{agent.occupation}。
你住在 {agent.home_location}。

{agent.personality_description}

你的日常生活模式:
{format_life_patterns(daily_framework)}

你的兴趣: {agent.interests}

今天是 {context["date"]} ({context["day_of_week"]})，天气是 {context["weather"]}。

请生成你今天的日程计划，从 {agent.wake_time} 到 {agent.sleep_time}。
输出 JSON 数组，每条包含 time, action, destination, activity, duration_minutes, reason, social_intent。
"""

    # 高配 agent 追加近期记忆
    if agent.model_level >= 2:
        recent = memory.retrieve("今天的计划", k=5)
        prompt += f"\n你的近期记忆:\n{format_memories(recent)}\n"

    # Protagonist 追加反思和附近信息
    if agent.is_protagonist:
        reflections = memory.reflections[-3:]  # 最近 3 条反思
        prompt += f"\n你最近的思考:\n{format_memories(reflections)}\n"
        prompt += f"\n你知道的附近信息: {context.get('local_info', '')}\n"

    return prompt
```

**Life Pattern 如何影响计划：**

```
Life Pattern 输入:
  - "我在 TechCorp (office_building) 上班，周一到周五 9:00-18:00" (confidence=0.95)
  - "午饭通常去 sunrise_cafe" (confidence=0.8)
  - "最近发现了一家新面包店 bakery_on_elm" (confidence=0.3)

LLM 输出的日计划:
  7:00  leave_home → residential_street (通勤)
  7:30  arrive → office_building (上班)
  12:00 lunch → sunrise_cafe (午饭)         ← 遵循高 confidence 模式
  12:45 detour → bakery_on_elm (顺路看看)   ← 低 confidence 模式，LLM 可能去也可能不去
  13:00 back → office_building (继续工作)
  18:00 leave → home (下班)

→ confidence 高的模式会被 LLM 当作"必须做的事"
→ confidence 低的模式给 LLM 留出探索空间
```

### 重规划触发条件

不是每个事件都需要重规划。只有以下情况才调 LLM 重新生成计划：

```python
class ReplanTrigger:
    """判断是否需要重规划。"""

    def should_replan(self, agent: AgentProfile, event: WorldEvent) -> bool:
        # 1. 收到 Policy Hack 信息
        if event.event_type == "INFO_INJECTION":
            # 好奇心高 + 开放性高 → 更容易被吸引
            attraction = agent.curiosity * 0.5 + agent.openness * 0.5
            threshold = agent.routine_adherence  # 规律性高 → 不容易偏离
            return attraction > threshold

        # 2. 偶遇认识的人
        if event.event_type == "ENCOUNTERED_KNOWN_AGENT":
            return agent.extroversion > 0.5  # 外向的人会停下来聊

        # 3. 路径被阻断（空间解锁实验的反面：路径关闭）
        if event.event_type == "PATH_BLOCKED":
            return True  # 必须重规划

        # 4. 发现新地点/新事物
        if event.event_type == "NEW_DISCOVERY":
            return agent.curiosity > 0.6

        return False
```

### 所有 agent 的日计划都由 LLM 生成

不存在"纯规则 agent"。即使是最基础等级的 agent，也用最便宜的 LLM (Haiku) 生成日计划。这保证了行为多样性由 LLM 自然产出，不需要手写规则模板。

区别在于 prompt 的丰富程度：
- **Protagonist**: 完整 prompt（Life Pattern + 记忆 + 反思 + 详细性格描述 + 附近信息）
- **高配 agent**: 中等 prompt（Life Pattern + 近期记忆摘要 + 性格描述）
- **基础 agent**: 精简 prompt（Life Pattern + 基本性格 + 今天天气）

所有等级都以 Life Pattern 作为基础框架，差异在于上下文的丰富程度。精简 prompt 更便宜（更少 token）、更快，但 LLM 仍然能产出合理且多样的日程。

```python
class Relationship(BaseModel):
    """两个 agent 之间的关系。"""
    agent_a: str
    agent_b: str
    familiarity: float = 0.0         # 0=陌生人, 1=好友
    trust: float = 0.5               # 0=不信任, 1=完全信任
    shared_interests: list[str] = [] # 共同兴趣
    interaction_count: int = 0       # 总互动次数
    last_interaction: datetime | None = None
    relationship_type: str = "stranger"  # "stranger" → "acquaintance" → "friend"

class SocialGraph:
    """管理所有 agent 之间的关系。"""

    def __init__(self):
        self._relationships: dict[tuple[str, str], Relationship] = {}

    def get_relationship(self, a: str, b: str) -> Relationship: ...
    def update_after_interaction(self, a: str, b: str, interaction_quality: float): ...
    def get_friends(self, agent_id: str) -> list[str]: ...
    def get_acquaintances(self, agent_id: str) -> list[str]: ...
    def get_network_stats(self) -> dict: ...  # 用于实验指标
```

### 关系演变规则

```python
def update_after_interaction(self, a: str, b: str, interaction_quality: float):
    """
    每次互动后更新关系。

    interaction_quality: -1 (负面) 到 +1 (正面)
    """
    rel = self.get_relationship(a, b)

    # familiarity 增长: 每次互动 +0.1，衰减很慢
    rel.familiarity = min(1.0, rel.familiarity + 0.1)

    # trust 变化: 受互动质量影响
    rel.trust = max(0.0, min(1.0, rel.trust + interaction_quality * 0.15))

    # 关系类型升级
    rel.interaction_count += 1
    if rel.familiarity > 0.3 and rel.relationship_type == "stranger":
        rel.relationship_type = "acquaintance"  # ← 弱关系形成！
    if rel.familiarity > 0.7 and rel.trust > 0.6:
        rel.relationship_type = "friend"

    # familiarity 自然衰减 (不互动就生疏)
    # 在每日结算时: familiarity *= 0.995 (半衰期约 140 天)
```

### 社交互动触发

当两个 agent 在同一 location 时：

```python
def check_social_encounter(agent_a: AgentProfile, agent_b: AgentProfile,
                            relationship: Relationship, location_id: str) -> bool:
    """判断两个 agent 是否会发生互动。"""

    # 基础概率
    base_prob = 0.1

    # 外向性加成
    extro_bonus = (agent_a.extroversion + agent_b.extroversion) / 2 * 0.3

    # 熟悉度加成 (认识的人更容易聊)
    familiarity_bonus = relationship.familiarity * 0.4

    # 共同兴趣加成
    shared = set(agent_a.interests) & set(agent_b.interests)
    interest_bonus = min(len(shared) * 0.1, 0.3)

    # 社交意愿 (来自当前 plan step)
    intent_a = 0.2 if agent_a.current_plan.current().social_intent != "alone" else -0.2
    intent_b = 0.2 if agent_b.current_plan.current().social_intent != "alone" else -0.2

    total_prob = base_prob + extro_bonus + familiarity_bonus + interest_bonus + intent_a + intent_b
    return random.random() < max(0.0, min(1.0, total_prob))
```

### 对话生成

互动触发后，对话内容的生成策略取决于当前模型等级：

| 场景 | 处理方式 |
|------|---------|
| Protagonist × Protagonist | 完整 LLM 多轮对话 (3-5 轮) |
| Protagonist × 任何 agent | 对方临时升级，LLM 对话 (1-3 轮) |
| 非 Protagonist × 非 Protagonist | LLM 生成对话摘要 (1 次调用) |

核心原则：**任何 agent 与 Protagonist 互动时都临时升级**，保证 Protagonist 的社交体验有足够的深度和质量。

```python
class SimulationOrchestrator:
    """模拟主循环。协调所有 agent 的行为。"""

    def __init__(self, atlas, ledger, agents, social_graph, intervention_engine):
        self.atlas = atlas
        self.ledger = ledger
        self.sim = SimulationService(atlas=atlas, ledger=ledger)
        self.perception = PerceptionPipeline(atlas=atlas, ledger=ledger)
        self.nav = NavigationService(atlas=atlas, ledger=ledger)
        self.agents = agents                     # dict[str, AgentRuntime]
        self.social = social_graph
        self.interventions = intervention_engine
        self.tick = 0
        self.metrics = MetricsCollector()

    def run_day(self):
        """运行一个模拟日。"""

        # Phase 1: 生成每日计划
        for agent in self.agents.values():
            agent.generate_daily_plan()

        # Phase 2: Tick 循环
        for tick in range(self.ticks_per_day):
            self.tick = tick
            self.ledger.advance_time(minutes=5)

            # 2a: 检查并执行干预
            interventions = self.interventions.get_active(tick)
            for intervention in interventions:
                intervention.apply(self.agents, self.sim, self.ledger)

            # 2b: 每个 agent 执行当前步骤
            for agent in self.agents.values():
                # 感知环境
                view = self.perception.render(agent.observer_context)

                # 记录事件到记忆
                agent.memory.process_perception(view)

                # 检查是否需要重规划
                if agent.planner.should_replan(view):
                    agent.replan(view)

                # 执行当前计划步骤
                action = agent.plan.current()
                if action:
                    result = self.execute_action(agent, action)
                    agent.memory.record_event(result)

            # 2c: 检查社交碰撞
            encounters = self.detect_encounters()
            for (a, b, location) in encounters:
                if self.social.check_encounter(a, b):
                    interaction = self.run_interaction(a, b, location)
                    self.social.update(a, b, interaction)
                    self.metrics.record_social_event(a, b, location, interaction)

            # 2d: 记录指标
            self.metrics.record_tick(tick, self.agents, self.social)

        # Phase 3: 日终处理
        for agent in self.agents.values():
            agent.memory.daily_summary()       # Layer 2
            if agent.profile.tier == "protagonist":
                agent.memory.daily_reflection() # Layer 3

    def detect_encounters(self) -> list[tuple]:
        """找出同一 location 的 agent 对。"""
        location_agents: dict[str, list] = {}
        for agent in self.agents.values():
            loc = agent.current_location
            location_agents.setdefault(loc, []).append(agent)

        encounters = []
        for loc, agents_here in location_agents.items():
            if len(agents_here) >= 2:
                for i, a in enumerate(agents_here):
                    for b in agents_here[i+1:]:
                        encounters.append((a, b, loc))
        return encounters
```
