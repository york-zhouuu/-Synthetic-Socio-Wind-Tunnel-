## Policy Hack 干预系统

**文件**: `synthetic_socio_wind_tunnel/agent/intervention.py`

Policy Hack 是模拟的核心实验变量——通过数字手段打破社交边界。

### 五种干预类型

```python
class InterventionType(str, Enum):
    INFO_INJECTION = "info_injection"         # 信息注入
    SPACE_UNLOCK = "space_unlock"             # 空间解锁
    OBJECT_PLACEMENT = "object_placement"     # 物品放置
    TASK_ASSIGNMENT = "task_assignment"        # 任务赋予
    PERCEPTION_MOD = "perception_modification" # 感知修改
```

### 干预的数据模型

```python
class Intervention(BaseModel):
    """一次干预实验。"""
    intervention_id: str
    intervention_type: InterventionType

    parameters: dict                   # 类型特定参数

    def apply(self, agents, simulation, ledger): ...
```

### 各类型的具体实现

#### 1. 信息注入 (Info Injection)

向 agent 推送超在地信息，模拟"数字诱饵"实验。

```python
class InfoInjection:
    """向范围内的 agent 推送本地信息。"""

    parameters = {
        "message": "距离你50米的小酒馆今晚有神秘试饮活动",
        "radius": 500,              # 推送范围 (米)
        "center_location": "sunset_bar",  # 信息关联的地点
        "urgency": 0.7,             # 紧迫感 (影响 agent 是否重规划)
        "category": "event",        # "event" / "discovery" / "social" / "alert"
    }

    def apply(self, agents, simulation, ledger):
        # 1. 找出范围内的所有 agent
        nearby = [a for a in agents if distance(a.location, self.center) < self.radius]

        # 2. 向每个 agent 注入信息
        for agent in nearby:
            # 添加到 agent 的感知中
            agent.receive_info(InfoEvent(
                content=self.message,
                source="phone_notification",  # 模拟手机推送
                related_location=self.center_location,
            ))
            # agent 的 Planner 会在下一个 tick 检查这个信息
            # 是否重规划取决于 agent 的 curiosity 和 routine_adherence
```

#### 2. 空间解锁 (Space Unlock)

打开一条新路径或开放一个空间，模拟"空间解锁"实验。

```python
class SpaceUnlock:
    """添加或解锁一条路径。"""

    parameters = {
        "connection_from": "block_a",
        "connection_to": "block_b",
        "path_type": "footway",
        "description": "围墙上新开了一扇小门",
    }

    def apply(self, agents, simulation, ledger):
        # 1. 在 Atlas 中添加新 Connection (或解锁已有的 locked door)
        # 注意: Atlas 是 frozen 的，这里需要通过 Ledger 记录"解锁状态"
        ledger.unlock_path(self.connection_from, self.connection_to)

        # 2. 让附近的 agent 感知到变化
        nearby = get_agents_near([self.connection_from, self.connection_to])
        for agent in nearby:
            agent.receive_info(InfoEvent(
                content=self.description,
                source="observation",
            ))
```

#### 3. 物品放置 (Object Placement)

在空间中放置社交催化物。

```python
class ObjectPlacement:
    """在某个位置放置物品作为社交催化。"""

    parameters = {
        "location": "dead_end_alley",
        "object_name": "公共钢琴",
        "object_type": "social_catalyst",
        "attraction_radius": 100,
        "description": "死胡同里出现了一架公共钢琴，任何人都可以弹",
    }

    def apply(self, agents, simulation, ledger):
        # 1. 在 location 放置物品
        simulation.place_item(
            item_id=f"catalyst_{self.object_name}",
            name=self.object_name,
            location_id=self.location,
        )
        # 2. 物品会被经过的 agent 感知到 (通过 PerceptionPipeline)
        # 3. 好奇心高的 agent 可能会停下来互动
```

#### 4. 任务赋予 (Task Assignment)

给多个 agent 共同的隐藏任务。

```python
class TaskAssignment:
    """给多个 agent 分配共同任务。"""

    parameters = {
        "task_description": "有只猫在附近走丢了，大家都在找",
        "assigned_agents": ["emma", "bob", "chen"],  # 或 None = 随机选
        "search_area": ["central_park", "main_street", "back_alley"],
        "shared_goal": True,          # 是否知道其他人也在做同样的事
    }

    def apply(self, agents, simulation, ledger):
        for agent_id in self.assigned_agents:
            agent = agents[agent_id]
            agent.receive_task(Task(
                description=self.task_description,
                search_locations=self.search_area,
                is_shared=self.shared_goal,
            ))
            # agent 的 Planner 会将任务整合进日程
            # 如果 shared_goal=True，agent 会主动寻找也在执行任务的人
```

#### 5. 感知修改 (Perception Modification)

改变 agent 观察世界的方式。

```python
class PerceptionModification:
    """修改 agent 的 ObserverContext。"""

    parameters = {
        "target_agents": ["emma", "bob", "chen"],
        "modifications": {
            "looking_for": ["lost_cat_poster", "paw_prints"],
            "attention": 0.9,
            "emotional_state": {"curiosity": 0.8, "empathy": 0.7},
        }
    }

    def apply(self, agents, simulation, ledger):
        for agent_id in self.target_agents:
            agent = agents[agent_id]
            # 修改 ObserverContext
            agent.observer_context.looking_for = self.modifications["looking_for"]
            agent.observer_context.attention = self.modifications["attention"]
            # 这直接影响 PerceptionPipeline 的输出
            # agent 会开始"注意到"之前忽略的细节
```

## 实验框架

### 实验配置

```python
class Experiment(BaseModel):
    """一次完整的实验配置。"""
    experiment_id: str
    name: str
    description: str

    # 时间线
    baseline_days: int = 3           # 干预前的基线观察天数
    intervention_day: int = 4        # 干预开始日
    observation_days: int = 7        # 干预后的观察天数

    # 干预列表
    interventions: list[Intervention]

    # 对照组
    control_group: list[str]         # 不受干预的 agent IDs
    treatment_group: list[str]       # 受干预的 agent IDs

    # 观测指标
    metrics: list[str]               # 要追踪的指标名称
```

### 实验执行流程

```
Day 1-3:  BASELINE (基线)
          ├── 所有 agent 正常运行，无干预
          ├── 记录轨迹、社交、空间使用
          └── 建立基线数据

Day 4:    INTERVENTION (干预)
          ├── 激活 Policy Hack
          ├── 标记为实验转折点
          └── 开始差异追踪

Day 5-10: OBSERVATION (观察)
          ├── 继续运行，记录变化
          ├── 对比 treatment vs control
          └── 收集叙事数据
```
