# 实操指南：从真实社区到可运行地图

## 前置条件

- Python 3.11+
- synthetic_socio_wind_tunnel 已安装（`pip install -e ".[full]"`）
- Node.js（用于 osmtogeojson 工具）
- Anthropic API key（用于 LLM 充实，Stage 2）

## Step 1: 选址与数据获取

### 1.1 选址标准

目标社区应满足：
- **封闭性** — 有明确的地理边界（河流、铁路、主干道等）
- **自循环** — 社区内有居住、商业、公共空间，居民日常需求可在内部解决
- **人群均匀** — 不是只有一个商业中心的辐射状布局
- **OSM 数据质量** — 建筑和道路数据要相对完整

### 1.2 检查 OSM 数据质量

在选定区域之前，先在 OpenStreetMap 上浏览检查：

1. 打开 https://www.openstreetmap.org
2. 导航到候选区域
3. 放大到建筑级别
4. 检查：
   - 建筑轮廓是否画出来了？
   - 建筑是否有名称和功能标签？（点击建筑查看属性）
   - **道路网络是否完整？**（重点：是否包含住宅区小路和人行道）
   - 公园、广场等公共空间是否标注？

**数据质量评分参考：**

| 等级 | 描述 | 能否用 |
|------|------|--------|
| A | 建筑有轮廓+标签，道路网完整含人行道 | 直接用 |
| B | 建筑有轮廓但缺标签，主要道路有 | 可用，LLM 充实补标签，可能缺少小路 |
| C | 只有主干道，建筑/小路缺失 | 不推荐，地图太稀疏 |

### 1.3 导出 GeoJSON

1. 在 OpenStreetMap 上框选目标区域
2. 点击 "Export" → "Overpass API"（如果区域太大）
3. 下载 `.osm` 文件
4. 转换为 GeoJSON：

```bash
npm install -g osmtogeojson
osmtogeojson community.osm > community.geojson
```

### 1.4 检查导出数据

```python
import json

with open("community.geojson") as f:
    data = json.load(f)

features = data.get("features", [])
print(f"总 feature 数: {len(features)}")

# 分类统计
buildings = [f for f in features if "building" in f.get("properties", {})]
roads = [f for f in features
         if f.get("geometry", {}).get("type") == "LineString"
         and "highway" in f.get("properties", {})]
areas = [f for f in features
         if any(k in f.get("properties", {}) for k in ["leisure", "landuse"])]

print(f"建筑: {len(buildings)}")
print(f"道路: {len(roads)}")
print(f"公共区域: {len(areas)}")

# 道路类型分布
from collections import Counter
road_types = Counter(r["properties"].get("highway") for r in roads)
print(f"道路类型分布: {dict(road_types)}")

# 前几个建筑
for b in buildings[:5]:
    props = b["properties"]
    print(f'  - {props.get("name", "无名")} | '
          f'building={props.get("building")} '
          f'amenity={props.get("amenity", "-")}')
```

**预期输出示例：**
```
总 feature 数: 342
建筑: 87
道路: 45
公共区域: 12
道路类型分布: {'residential': 22, 'footway': 10, 'service': 8, 'secondary': 3, 'path': 2}
  - Sunrise Café | building=commercial amenity=cafe
  - 无名 | building=residential amenity=-
  - Community Library | building=yes amenity=library
```

**关键检查：**
- 建筑数 < 20 → 数据太少
- 道路数 < 10 → 路网不完整，agent 移动路径会不自然
- 如果没有 `footway`/`path` 类型 → 缺少人行道，只有车道

## Step 1.5: （可选）多源富化

默认 Pipeline 仅消费 OSM。`enrich-lanecove-map` change 在 OSM 之上引入
Overture Maps 的 Buildings + Places 富化层，为建筑补齐具名、分类、高度、POI
等信息。流程：

```
┌───────────────────────┐
│ lanecove_osm.geojson  │  ←  fetch_lanecove_v2.py（已有）
└───────────┬───────────┘
            │
            │  ┌─────────────────────────────────┐
            │  │ overture_buildings.geojson      │
            │  │ overture_places.geojson         │  ←  tools/fetch_overture.py
            │  └─────────────────────────────────┘
            ▼                   │
     ┌────────────────────────────────┐
     │ cartography/conflation.py      │  合并：OSM 保名 + Overture 补字段
     │ merge_sources(...)             │           + Places → affordances
     └──────────────┬─────────────────┘
                    ▼
     ┌────────────────────────────────┐
     │ data/lanecove_enriched.geojson │
     └──────────────┬─────────────────┘
                    ▼
            GeoJSONImporter ── Atlas
```

命令序列：

```bash
# 1. 拉取 Overture（需要网络 + 预装 overturemaps CLI: pip install -e ".[map-enrichment]"）
python3 tools/fetch_overture.py

# 2. 与 OSM 合并成一个富化 GeoJSON
python3 tools/enrich_map.py
# → 产物: data/lanecove_enriched.geojson, data/lanecove_enriched.stats.json

# 3. 清除旧 atlas 缓存，让 create_atlas_from_osm 用富化源重建
rm data/lanecove_atlas.json
python3 -c "from tools.map_explorer.mock_map import create_atlas_from_osm; create_atlas_from_osm()"

# 4. 诊断：看具名率 / 类型分布 / affordance 覆盖
python3 tools/diagnose_atlas.py
```

若不跑富化步骤，`mock_map` 会自动回退到纯 OSM，接口不变。

## Step 2: 运行 Pipeline

### 2.1 Stage 1 — OSM 提取

```python
from synthetic_socio_wind_tunnel.cartography.importer import GeoJSONImporter

importer = GeoJSONImporter()
skeleton = importer.import_file(
    "community.geojson",
    scale=10000.0,
    region_id="my_community",
    segment_length=80.0,           # 街道段长度 (米)
    building_snap_distance=30.0,   # 建筑连接到街道的最大距离
)

# 检查结果
buildings = skeleton.buildings
streets = {k: v for k, v in skeleton.outdoor_areas.items() if v.area_type == "street"}
open_spaces = {k: v for k, v in skeleton.outdoor_areas.items() if v.area_type != "street"}

print(f"建筑: {len(buildings)}")
print(f"街道段: {len(streets)}")
print(f"公共空间: {len(open_spaces)}")
print(f"连接: {len(skeleton.connections)}")

# 按道路名分组查看街道段
from collections import defaultdict
roads = defaultdict(list)
for seg in streets.values():
    roads[seg.road_name].append(seg)
for road_name, segs in sorted(roads.items()):
    print(f"  {road_name}: {len(segs)} 段")
```

**预期输出：**
```
建筑: 87
街道段: 156
公共空间: 12
连接: 412

  Elm Street: 8 段
  Main Street: 12 段
  Oak Avenue: 6 段
  Park Lane: 4 段
  ...
```

### 2.2 验证地图连通性

```python
from synthetic_socio_wind_tunnel import Atlas, Ledger, NavigationService

atlas = Atlas(skeleton)
ledger = Ledger()
nav = NavigationService(atlas=atlas, ledger=ledger)

# 取两个最远的建筑，测试能否寻路
building_ids = list(buildings.keys())
start, end = building_ids[0], building_ids[-1]

route = nav.find_path(start, end)
if route.success:
    print(f"寻路成功: {start} → {end}")
    print(f"  步数: {len(route.steps)}")
    print(f"  总距离: {route.total_distance:.1f}")
    print(f"  路径:")
    for step in route.steps:
        loc = skeleton.outdoor_areas.get(step.location_id) or skeleton.buildings.get(step.location_id)
        loc_type = "street" if hasattr(loc, "area_type") and loc.area_type == "street" else "location"
        print(f"    → {step.location_id} ({loc_type})")
else:
    print(f"寻路失败! 地图可能不连通")
    # 检查孤立节点
    connected_ids = set()
    for conn in skeleton.connections:
        connected_ids.add(conn.from_id)
        connected_ids.add(conn.to_id)
    all_ids = set(buildings.keys()) | set(skeleton.outdoor_areas.keys())
    isolated = all_ids - connected_ids
    print(f"孤立节点: {isolated}")
```

**预期输出：**
```
寻路成功: sunrise_cafe → maple_apartments
  步数: 8
  总距离: 420.5
  路径:
    → sunrise_cafe (location)
    → main_st_seg_2 (street)        ← agent 走上街道
    → main_st_seg_3 (street)
    → main_st_seg_4 (street)
    → elm_st_seg_1 (street)         ← 在十字路口转弯
```

### 2.5 Stage 3 — 人工审核

打开 `atlas.json`，重点检查：

```python
# 1. 检查孤立节点
connected_ids = set()
for conn in atlas_data["connections"]:
    connected_ids.add(conn["from_id"])
    connected_ids.add(conn["to_id"])
all_ids = set(atlas_data["buildings"].keys()) | set(atlas_data["outdoor_areas"].keys())
isolated = all_ids - connected_ids
if isolated:
    print(f"孤立节点 (无连接): {isolated}")
else:
    print("所有节点都有连接")

# 2. 检查街道段的连续性
from collections import defaultdict
road_segs = defaultdict(list)
for area_id, area in atlas_data["outdoor_areas"].items():
    if area.get("area_type") == "street":
        road_segs[area.get("road_name", "")].append(area)

for road_name, segs in road_segs.items():
    segs.sort(key=lambda s: s.get("segment_index", 0))
    # 检查相邻段之间是否有 connection
    for i in range(len(segs) - 1):
        seg_a, seg_b = segs[i]["id"], segs[i+1]["id"]  # use 'id' field name
        has_conn = any(
            (c["from_id"] == seg_a and c["to_id"] == seg_b) or
            (c["from_id"] == seg_b and c["to_id"] == seg_a)
            for c in atlas_data["connections"]
        )
        if not has_conn:
            print(f"断裂: {road_name} 的 {seg_a} 和 {seg_b} 之间没有连接!")

# 3. 检查每个建筑是否连到街道
for b_id in atlas_data["buildings"]:
    has_street_conn = any(
        (c["from_id"] == b_id and "seg" in c["to_id"]) or
        (c["to_id"] == b_id and "seg" in c["from_id"])
        for c in atlas_data["connections"]
    )
    if not has_street_conn:
        print(f"建筑 {b_id} 没有连接到任何街道段")
```

## Step 3: 加载验证

### 3.1 完整加载测试

```python
import json
from synthetic_socio_wind_tunnel import Atlas, Ledger, SimulationService, NavigationService

with open("atlas.json") as f:
    atlas_data = json.load(f)

atlas = Atlas.from_dict(atlas_data)
ledger = Ledger()
sim = SimulationService(atlas=atlas, ledger=ledger)
nav = NavigationService(atlas=atlas, ledger=ledger)

buildings = atlas.region.buildings
streets = {k: v for k, v in atlas.region.outdoor_areas.items() if v.area_type == "street"}

print(f"地图加载成功: {atlas.region.name}")
print(f"  {len(buildings)} 建筑, {len(streets)} 街道段, {len(atlas.region.connections)} 连接")
```

### 3.2 测试逐步移动

模拟 agent 按寻路结果逐步移动（每 tick 一步）：

```python
# 创建 agent 在起始建筑
start = list(buildings.keys())[0]
end = list(buildings.keys())[-1]
ledger.set_entity("test_agent", location_id=start)

# 寻路
route = nav.find_path(start, end)
print(f"路线: {start} → {end}, {len(route.steps)} 步")

# 逐步移动
for i, step in enumerate(route.steps):
    result = sim.move_entity("test_agent", to_location=step.location_id)
    loc = atlas.region.outdoor_areas.get(step.location_id)
    is_street = loc and loc.area_type == "street"
    print(f"  tick {i}: → {step.location_id} {'(street)' if is_street else ''} {'OK' if result.success else 'FAIL'}")
```

### 3.3 测试路上碰面

```python
# 创建两个 agent
ledger.set_entity("emma", location_id=list(buildings.keys())[0])
ledger.set_entity("bob", location_id=list(buildings.keys())[-1])

# 各自寻路到对方的位置（迎面走来）
route_emma = nav.find_path(list(buildings.keys())[0], list(buildings.keys())[-1])
route_bob = nav.find_path(list(buildings.keys())[-1], list(buildings.keys())[0])

# 逐步移动，检查是否在同一街道段上
for tick in range(max(len(route_emma.steps), len(route_bob.steps))):
    if tick < len(route_emma.steps):
        sim.move_entity("emma", route_emma.steps[tick].location_id)
    if tick < len(route_bob.steps):
        sim.move_entity("bob", route_bob.steps[tick].location_id)

    emma_loc = ledger.get_entity("emma").location_id
    bob_loc = ledger.get_entity("bob").location_id

    if emma_loc == bob_loc:
        print(f"  tick {tick}: Emma 和 Bob 在 {emma_loc} 碰面了!")
```

## 常见问题

### Q: GeoJSON 文件太大怎么办？
用 bounds 参数裁剪：
```python
skeleton = importer.import_file("big.geojson", bounds={
    "min_lat": -33.91, "max_lat": -33.90,
    "min_lon": 151.20, "max_lon": 151.21,
})
```

### Q: 街道段太多导致地图太大？
增加 `segment_length`：
```python
skeleton = importer.import_file("community.geojson", segment_length=120.0)  # 每段 120m
```

### Q: 某些建筑没有连到街道？
降低 `building_snap_distance` 不会帮助（那会更严格）。应该增大它：
```python
skeleton = importer.import_file("community.geojson", building_snap_distance=60.0)  # 60m 范围
```

### Q: OSM 数据中建筑没有名字怎么办？
importer 会自动生成 `building_0`, `building_1` 等 ID。LLM enricher 可以根据功能类型生成更好的名字。

### Q: scale 参数怎么选？
默认 10000 将经纬度差值放大到模拟坐标。一般来说：
- 小社区（< 1km²）: scale=10000
- 中等区域（1-5km²）: scale=5000
- 大区域（> 5km²）: scale=2000

### Q: Connection 推断不准，可以手动调吗？
可以。导出 atlas.json 后直接编辑 connections 数组：
```json
{"from_id": "sunrise_cafe", "to_id": "main_st_seg_2", "distance": 10.0, "path_type": "entrance"}
```
